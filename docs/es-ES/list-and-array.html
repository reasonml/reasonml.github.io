Lista y Matriz

Lista

Las listas son:

  • homogéneas
  • inmutables
  • rápidas al anteponer items
let myList = [1, 2, 3];

Las listas de Reason son simples, listas individualmente enlazadas.

Uso

Usaras listas por su capacidad de ser redimensionadas, su rapidez anteponiendo (añadiendo al inicio), y su rápida division, ¡características inmutables pero eficientes!

La biblioteca estándar proporciona un módulo de Lista (y su complemento con argumentos etiquetados, ListLabels).

Anteposición Inmutable

Utiliza la sintaxis de expansión, que es simplemente List.cons:

let myList: list(int) = [1, 2, 3];
let anotherList = [0, ...myList];

myList no mutó. anotherList is now [0, 1, 2, 3]. Esto es eficiente (tiempo constante, no lineal). Los 3 últimos elementos de anotherListson compartidos con myList!

Tenga en cuenta que [a, ...b, ...c] es un error de sintaxis. No soportamos múltiples expansiones para una lista. Sería una operación lineal accidental (O(b)), ya que cada elemento de b sería uno a uno añadido al comienzo de c. Puedes utilizar List.concat para ello.

Se desaconseja actualizar un elemento arbitrario en medio de una lista, ya que su rendimiento y la sobrecarga de asignación sería linear (O(n)).

Acceso

switch (descrito en la sección sobre coincidencia de patrón) es habitualmente usado para acceder a elementos de la lista:

let message =
  switch (myList) {
  | [] => "This list is empty"
  | [a, ...rest] => "The head of the list is the string " ++ string_of_int(a)
  };

Para acceder a un elemento arbitrario de la lista, utiliza List.nth.

Consejos y trucos

No dude en asignar tantas listas vacías como usted quisiera. Como se explica en la sección variante para lista, una lista vacía es realmente un constructor de variante sin parámetros bajo el capó, que compila a un simple entero. No es necesaria la asignación de memoria extra.

Para entender cómo la anteposición puede ser inmutable y O(1) al mismo tiempo, vea también el enlace anterior.

Decisiones de Diseño

En el futuro, podríamos proporcionar directamente una estructura de datos de tipo lista que sea inmutable, redimensionable y con todo tipo de operaciones rápidas y eficientes.

Array

Los arrays son como las listas, excepto que son:

  • mutables
  • de rápido acceso aleatorio y actualizaciones
  • de tamaño fijo en código nativo (y tamaño flexible en JavaScript)

Debes escribirlos entre [| y |].

let myArray = [|"hola", "mundo", "que tal estás"|];

Uso

Módulo Array y ArrayLabel de la biblioteca estándar. For JS compilation, you also have the familiar Js.Array API.

Acceder y actualizar un elemento de el array así:

let myArray: array(string) = [|"hello", "world", "how are you"|];

let firstItem = myArray[0]; /* "hello" */

myArray[0] = "hey";

/* now [|"hey", "world", "how are you"|] */

El acceso/actualización al array anterior es sólo endulzamiento sintáctico (sintaxis sugar) para Array.get/Array.set.

Consejos y trucos

Si estás compilando a JavaScript, que sepas que loas arrays de Reason mapean directamente a arrays de JavaScript y viceversa. Y, a pesar de que los arrays son de tamaño fijo en código nativo, todavía puedes utilizar Js.Array API para redimensionarlos. Esto es aceptable.
