Démarrage rapide en natif

esy

Esy (prononcé "easy") est un gestionnaire de paquets pour les applications natives en Reason. Il vous permet de consommer des paquets opam et de publier vos propres paquets natifs sur npm.

Installer esy :

npm install -g esy

Note : Assurez-vous que vous avez le dernier esy installé. Si vous avez un doute, exécutez npm remove -g esy && npm install -g esy@latest.

Utiliser esy :

Clonez n'importe quel projet esy et exécutez la commande esy à l'intérieur. Cela installera toutes les dépendances et compilera tout le nécessaire.

git clone git@github.com:esy-ocaml/hello-reason.git
cd hello-reason
esy

Ajouter des dépendances :

Pour ajouter des paquets Reason natifs qui se trouvent hébergés sur npm, exécutez esy add npm-package-name.

esy add refmterr

Intégration avec Opam :

esy traite le scope npm @opam spécialement. esy installera n'importe quel nom de paquet avec le scope @opam directement depuis opam. C'est le seul scope avec une signification particulière. Tous les autres noms de paquets sont présumés être sur npm.

esy add @opam/bos

Configuration avancée d'esy :

Voir la section configuration de la documentation d'esy.

Support des éditeurs :

Voir la section projet natif de la documentation de support des éditeurs pour Reason.

Compatibilité :

En plus de supporter les paquets spécifiques esy hébergés sur npm, esy supporte également tout paquet opam ainsi que le compilateur natif ocaml. Il peut aussi être utilisé pour développer des projets npm JavaScript qui sont compatibles avec Yarn Plug'n'Play (ce qui inclut la plupart des projets JavaScript). Cette issue Github suit les fonctionnalités requises pour permettre la gestion des projets BuckleScript en utilisant esy.
