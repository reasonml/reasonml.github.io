String & Char

String

Strings em Reason são delimitadas por aspas duplas (aspas simples são reservadas para o tipo char abaixo).

let greeting = "Hello world!";
let multilineGreeting = "Hello
 world!";

Caracteres especiais na string precisam ser escapados:

let oneSlash = "\\";

Para concatenar strings, utilize ++:

let greetings = "Olá " ++ "mundo!";

Quoted String

Há uma sintaxe especial para strings que permite:

  • strings de múltiplas linhas como anteriormente
  • falta de necessidade de escape de caracteres
  • hooks para pré-processadores especiais
let greetingAndOneSlash = {|Hello
World
\
Hehe...
|};

Analogicamente falando, é como a interpolação de backtick string do JavaScript, mas sem precisar escapar caracteres especiais e sem interpolação interna de variáveis. Embora você possa recuperar trivialmente a mesma funcionalidade como o BuckleScript tem feito:

let world = {js|世界|js}; /* Supports Unicode characters */
let helloWorld = {j|你好，$world|j}; /* Supports Unicode and interpolation variables */

O pré-processador especial de BuckleScript pode então procurar os marcadores js e j em torno da string e transformá-los em outra coisa.

Uso

Mais operações com strings podem ser encontradas na biblioteca padrão. For JS compilation, see the familiar Js.String API in the BuckleScript API docs. Uma vez que strings de Reason mapeiam para uma string de JavaScript, você pode misturar & combinar as operações de string em ambas as bibliotecas padrão.

Dicas & Truques

https://twitter.com/jusrin00/status/875238742621028355

Você possui um expressivo sistema de tipos agora! Em uma linguagem não tipada, muitas vezes você iria sobrecarregar o significado de string o utilizando como:

  • um id único: var BLUE_COLOR = "blue"
  • um identificador em uma estrutura de dados: var BLUE = "blue"; var RED = "red"; var colors = [BLUE, RED]
  • o nome de um campo em um objeto: pessoa["idade"] = 24
  • um enum: if (audio.canPlayType() === 'probably') {...} (ಠ_ಠ)
  • outros padrões malucos que em breve você achará horríveis, depois de se acostumar com as alternativas em Reason.

Quanto mais você sobrecarregar o tipo string (ou um coleguinha), menos o sistema de tipos poderá te ajudar! Reason fornece tipos & estruturas de dados alternativas concisos, rápidos e de fácil manutenção para os casos de uso acima (por exemplo, variantes, em uma seção posterior).

Sob a compilação nativa, a string de Reason compila para uma representação simples onde a performance é diretamente analisada, com custo de algumas vezes precisar de um tuning de performance manual. Por exemplo, concatenação nativa de strings como "hi" ++ "how" ++ "are" ++ "you?" desnecessariamente aloca as strings intermediárias "are you?" e "how are you?" (embora possa ser otimizado em uma única string nestes casos simples). Nesse caso, prefira String.concat. De certa forma, é um pouco mais legal que a análise que aprendemos na escola, talvez ela possa ser útil novamente.

Under JavaScript compilation, a Reason string maps to a JavaScript string and vice-versa, so no such above concern or analysis opportunities apply.

Decisões de Design

O recurso de quoted strings de não escapar caracteres especiais permite DSLs como expressões regulares:

let r = Str.regexp({|hello \([A-Za-z]+\)|});

em oposição a

let r = Str.regexp("hello \\([A-Za-z]+\\)");

Ao invés disso, ao compilar para JS, você deve usar [%bs.re] e Js.Re, já que Str não está disponível.

Reason/OCaml's foca na simplicidade em vez de algo demasiado complicado, isso pode ser visto diretamente através da implementação nativa de string. Uma implementação de string extremamente sofisticada, as vezes pode ser um retrocesso.

Char

Reason possui um tipo para uma string com apenas um caractere:

let firstLetterOfAlphabet = 'a';

Observação: Char não suporta Unicode ou UTF-8.

Dicas & Truques

Um caractere compila para um inteiro que varia de 0 a 255, para obter velocidade extra. Você também pode aplicar casamento de padrões (pattern matching, abordado mais tarde):

let isVowel = (theChar) =>
  switch (theChar) {
  | 'a' | 'e' | 'i' | 'o' | 'u' | 'y' => true
  | _ => false
  };

Para converter uma String para um Char, utilize "a".[0]. Para converter um Char para uma String, utilize String.make(1, 'a').
