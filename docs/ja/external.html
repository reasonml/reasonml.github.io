External

external、または、"FFI" (foreign function interface)、単に "interop"（相互運用性のための）は、Reason が他言語（C や JavaScript など）と通信する手段です。

Imagine writing a let binding, but with its body omitted and its (mandatory) type written down:

external myCFunction: int => string = "theCFunction";

[@bs.val] external getElementsByClassName: string => array(Dom.element) =
  "document.getElementsByClassName";

（上記は BuckleScript 固有の external で、同じ名前の JavaScript 関数に束縛されます。）

注意：externals はトップレベル、または、モジュール定義の内部でしか宣言することはできません。例えば関数の本体でそれらを宣言することはできません。

使い方

通常の let 束縛であるかのように external な値/関数を使用します。

ヒントとコツ

あなたが JavaScript から来た場合、BuckleScript externals について学ぶのに、いくらかの時間がかかります！ 最初に 100% 純粋な慣用的な Reason のコードを書く前に、かなりの数の externals に遭遇することになると思います。

設計方針

Reason は、非常に真摯に既存のコードと相互運用を受け入れます。 私たちの型システムはとても強力な保証があります。 しかし、このような強力な機能は、素晴らしい相互運用システムなしで、コードベースを徐々に Reason に変換していくことは非常に難しいであろうということも意味します。 幸い、FFI は 汚い、既存コード と非常にうまく協力することができます。 堅固な型システム + すばらしい相互運用の組み合わせは、私たちが漸進的型システムの欠点を除いて、インクリメンタルなコードベースカバレッジとコンバージョンについての伝統的な漸進的型システムの恩恵を受けられることを意味します。
