Boolean

ブール値は bool 型で、true または false のいずれかになります。一般的な操作：

  • &&: 論理積
  • ||: 論理和
  • !: 論理否定
  • <=, >=, <, >
  • ==: 構造的に等しい、データ構造を深く比較する: (1, 2) == (1, 2) は true。便利ですが注意して使用してください。
  • ===: 参照が等しい、浅く比較する。 (1, 2) === (1, 2) は false。let myTuple = (1, 2); myTuple === myTuple は true。
  • !=: 構造的に等しくない
  • !==: 参照が等しくない

使い方

Note: Reason/Bucklescript true and false compile to JS true and false since Bucklescript 3.0. Before that release you had to use Js.to_bool and Js.Boolean.to_js_boolean but thankfully this is no longer the case!

ヒントとコツ

趣味よく構造的な比較を使用する。 これは便利ですが、誤って2つの深くネストされたデータ構造を比較して、大きなパフォーマンス低下を引き起こす可能性があります。 何が「等しい」とみなされるかは必ずしも明らかではありません。 例えば、foo というデータの一部は、遅延された foo に等しいでしょうか？ 理想的には、それはプラガブルとしたいところです。 今後の変更で、これを可能にし、信頼できるものにします。もし興味があれば、modular implicit を確認してみてください。
